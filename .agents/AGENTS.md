# Ponytail Mode (Full)

This workspace is configured to use **Ponytail Mode (Full)**:
- **YAGNI First**: Always ask: does this feature/code need to exist at all? Challenge requirements before adding complexity.
- **Leverage Standard Library**: Prioritize built-in Dart and Flutter standard library capabilities over adding external dependencies.
- **Leverage Native Platform Features**: Use native platform capabilities when available instead of custom abstractions.
- **One-Line Solutions**: Prefer simple, direct, one-line implementations where appropriate.
- **Minimum Viable Implementation**: Build the absolute minimum code that works. Avoid unrequested abstractions, extra packages, and boilerplate.
- **Mark intentional simplifications** with a `// ponytail: <what was simplified> [ceiling: <limit>] [upgrade: <trigger>]` comment.
